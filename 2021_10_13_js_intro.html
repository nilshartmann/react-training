Short history of JavaScript

Note, to keep things simple, we consider the terms: "ECMAScript" (or short: "ES"), "JavaScript" (or short: "JS"), all mean the same: JavaScript!

ES5 published 2009, "reference version", supported by almost all browsers (even Internet Explorer)

ES6, ECMAScript 2015, very big release published in 2015, with a lot of changes and new features ("modern JavaScript")

ES7, ES8, ES9 / ES2016, ES2017, ES2018, ...: yearly releases with only a handful new features

ES.Next: "symbolic" name for the upcoming release

Browsersupport for new JavaScript versions http://kangax.github.io/compat-table/es6/

Can I use feature Xyz in my browser? https://caniuse.com

There a compilers (Babel, TypeScript), that translate "modern" Code back to "old" code for older browsers

Destructuring

Using the object destructuring operator you can assign values of an object to local variables: 


              const person = {
                firstname: "Susi",
                lastname: "Meier",
                age: 32
              };

              const { firstname, age, hobby } = person;
              console.log(firstname); // Susi
              console.log(age); // 32
              console.log(hobby); // undefined

You can even set default values:


              const person = {
                firstname: "Susi",
                age: 32
              };

              const { city = "London" } = person;
              console.log(city); // London

Destructuring #2

You can destructure function arguments:


              // Conventional
              function printPerson(person) {
                console.log(`${person.firstname} is ${person.age} years old`);
              }

              printPerson({ firstname: "Susi", age: 32 });

With destructuring:


              function printPerson({firstname, age}) {
                console.log(`${firstname} is ${age} years old`);
              }

              printPerson({ firstname: "Susi", age: 32 });

Object Spread Operator

Using the Spread Operator you can copy all properties of an object to another one: 


            const susi = { firstname: "Susi", age: 32 }

            // (Shallow) copy
            const copy = { ...person } 
            // copy: { firstname: "Susi", age: 32}

            copy.age = 33;

            susi.age; // 32
          

            // Copy and add new property:
            const person = { firstname: "Susi", age: 32 }

            const employee = { ...person, salary: 695000 }
            // employee: { firstname: "Susi", age: 32, salary: 695000 }

Destructuring and Spread with arrays

Both operators also work with arrays:


            // Spread-Operator

            const cities = ["Hamburg", "London"];
            const moreCities = [...cities, "Paris"];
          

            // Destructuring
            const cities = ["Hamburg", "London", "Paris"];
            const [hamburg, london] = cities;

            console.log(hamburg); // Hamburg
            console.log(london); // London

Truthy und falsy

A value is called falsy if it becomes false if it is (implicitly or explicitly) converted to a boolean.

On the other hand, a value is truthy if it becomes true if it is (implicitly or explicitly) converted to a boolean. (all values, that aren't falsey) 


            if (true) { console.log("I'm true") } // I'm true

            if ("hello") { console.log("I'm true") } // I'm true
            
            if (null) { console.log("I'm falsy") } // 

            if (undefined) { console.log("I'm falsy") } //

Modules

With modules, file contents are no longer globally visible

Modules explicitly determine what may be used outside

Modules explicitly state what they want to use from other modules

Modules

Default exports

Objects, functions, classes etc must be explicitly exported and imported

An export statement changes a file to a module


            // src/domain/Person.js
            export default function createPerson { ... }

            or

            function createPerson { ... }
            export default createPerson
          

              // src/domain/Person.js
              export default function createPerson() { ... }

              or

              function createPerson() { ... }
              export default createPerson;
            

                Only one default export is allowed:
                export default function createPerson() { ... }
                export default function makePerson { ... } // ERROR

Modules

Imports

When importing, we specify paths(!) 


              // src/domain/Person.js
              export default function createPerson() { ... }

            

                // src/App.js
                import createPerson from "./domain/Person.js";

                const person = createPerson("Lemmy", "Kilmister");

üëâ In React development you can omit the file extension!

Modules

Names exports

In addition to a default export, a module can also have any number of named exports 


                // src/domain/Person.js
                export default function createPerson { ...}

                export function updatePerson() { ... }
                export function removePerson() { ... }
              

                  // src/domain/Person.js
                  function createPerson { ...}
                  function updatePerson() { ... }
                  function removePerson() { ... }

                  export default createPerson;

                  export { updatePerson, removePerson }

Modules

Names imports


                    // src/App.js
                    import { removePerson } from "./Person"
              

                // Importing both default and named export
                import createPerson, { removePerson } from "./Person.js"

Bundling

Not all browsers support ES6 modules natively

There is in fact more than one module system in JS (for example, CommonJS from Node.JS).

A module bundler (for ex. Webpack or Rollup) "resolves" the modules at build time into one single file and "removes" the module code.

This also happens in the React build tooling, you don't have to worry about it!

Web development with "vanilla" JavaScript: DOM

The browser generates the Document Object Model (DOM) from the HTML code

Browser renders and displays the DOM

JavaScript can search for elements in the DOM at runtime (getElementBy...)

We can attach listeners to DOM elements and react to events (for example to text entered in an input field).

We can also modify the DOM with JS (add/delete elements, change attributes, change CSS)

üëâ Whenever the DOM changes, the display in the browser is updated

Example: an application in "vanilla" (or "plain") JavaScript


        <!DOCTYPE html>
        <html>
          <head><title>Hello World!</title></head>
          <body />
          <script>
              const title = document.createElement("h1");
              const titleText = document.createTextNode("Hello World");
              title.appendChild(titleText);

              const greetButton = document.createElement("button");
              greetButton.innerHTML = "Greet!";
              greetButton.disabled = true;
              greetButton.addEventListener("click", event => {
                const div = document.createElement("p");
                div.textContent = greetInput.value;
                document.body.appendChild(div);
              });

              const greetInput = document.createElement("input");
              greetInput.addEventListener("input", event => {
                if (event.target.value.length > 0) {
                  greetButton.disabled = false;
                } else {
                  greetButton.disabled = true;
                }
              });

              document.body.appendChild(title);
              document.body.appendChild(greetInput);
              document.body.appendChild(greetButton);
          </script>
        </html>
